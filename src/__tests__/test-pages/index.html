Time Is Money - Price Detection Test Pages

These test pages provide standardized examples of various price formats across different e-commerce sites and structural patterns. They serve as regression testing tools to ensure the Time Is Money extension maintains robust price detection capabilities across versions.

Each page contains examples with metadata about how they should be detected, which can be used to measure and validate the extension's detection accuracy.

Note: To use these test pages, open them in a browser with the Time Is Money extension installed. If detection debugging is enabled in the extension settings, you will see visual highlighting of detected prices.

Site-Specific Test Pages

  • Amazon Price Formats - Tests both sx-price-* and a-price-* patterns used across Amazon
  • eBay Price Formats - Tests various eBay-specific price structures
  • General E-commerce Formats - Common patterns from Shopify, WooCommerce, and other platforms

Structure-Specific Test Pages

  • Composite Price Structures - Tests prices split across multiple elements
  • Attribute-Based Prices - Tests prices defined by attributes rather than text content

Verification

Each test page includes metadata for price examples that indicates:

  • data-expect-detect: "true" if the price should be detected, "false" otherwise
  • data-detector: Which detector should identify this price ("amazon", "ebay", "attribute", "generic")
  • data-price-value: The expected numeric value that should be extracted
  • data-currency: The expected currency that should be detected

Use the Verify Detection button on each page to check detection success rates.
